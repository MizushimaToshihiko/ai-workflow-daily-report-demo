# 架空の日報CSV活動履歴分割ツール

このツールは、営業日報などの一覧CSVデータ（`daily_report.csv`）を読み込み、顧客コード（`customer_code`）ごとに活動履歴を分割して個別CSVファイルとして `output` フォルダに出力するPythonスクリプトです。

Pythonの標準ライブラリのみを使用しており、外部ライブラリをインストールすることなく実行可能です。

## 動作環境

- Python 3.x

## 実行方法

プロジェクトのルートディレクトリで以下のコマンドを実行します。

```bash
python main.py
```

## 入力CSV仕様 (`daily_report.csv`)

### 入力形式
- 文字コード: `utf-8-sig`
- カンマ区切りのCSV形式

### 必須列（ヘッダー）
以下の6列がすべて存在している必要があります。

| 列名 | 内容 |
| :--- | :--- |
| `date` | 日報の日付（`YYYY-MM-DD` 形式を想定） |
| `staff` | 担当者名 |
| `customer_code` | 顧客コード（先頭ゼロを含む文字列。例: `0001`） |
| `customer_name` | 顧客名 |
| `activity` | 活動内容 |
| `next_action` | 次回対応内容 |

---

## 出力CSV仕様

### 出力先フォルダ
- `output` フォルダ（存在しない場合は自動的に作成されます）。
- 既存の `output` フォルダ内に同名の出力ファイルがある場合は上書きされますが、今回の処理対象外である古いCSVファイルは削除されず維持されます。

### 出力ファイル名
- `customer_顧客コード.csv`（例: `customer_0001.csv`）
- ※ `customer_code` の先頭のゼロ（`0001`等）は文字列として維持されます。

### 出力列
入力CSVと同様に、以下の順番で出力されます。
`date,staff,customer_code,customer_name,activity,next_action`

### ソート順
各ファイル内のデータは、`date`（日付）の**昇順**で並べ替えられて出力されます。

---

## 除外条件

以下のいずれかに該当するデータ行は、活動履歴として不完全であるため、出力対象から除外されます。

1. **`customer_code` が空欄、または空白文字（半角スペース・全角スペース）のみの行**
2. **`activity` が空欄、または空白文字（半角スペース・全角スペース）のみの行**

---

## エラー時および除外時の挙動

### 1. `daily_report.csv` が存在しない場合
処理を実行せず、コンソールに以下のエラーメッセージを表示してプログラムを終了します。
```text
daily_report.csv が見つかりません。
```

### 2. 必須列（ヘッダー）が不足している場合
CSVのヘッダー行に必要な列（`date`, `staff`, `customer_code`, `customer_name`, `activity`, `next_action`）が含まれていない場合、処理を中断し、不足している列名をコンソールに表示してプログラムを終了します。
```text
必須列が不足しています: customer_code, activity
```

### 3. データ行が除外された場合
除外条件に該当する行は出力対象から除外され、処理終了時にコンソールに除外された件数として表示されます。プログラムは異常終了せず、有効なデータのみで処理を完了します。

実行完了時には以下のような集計結果がコンソールに表示されます。
```text
読み込んだ総行数: 14
出力対象となった行数: 10
除外された行数: 4
作成したCSVファイル数: 4
```
